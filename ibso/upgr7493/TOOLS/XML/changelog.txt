------------------------------------------------------------------------
v1.4.0.1

[-] BFS00036047, BFS00035697, PLTM7107
Исправлена ошибка работы с памятью в процедуре parseCLOB.


v1.4

[+] Реализованы следующие функции:
		setValidationMode/getValidationMode(установка/получение
		 режима проверки документа по DTD, по умолчанию проверка отключена)
		encodeBase64/decodeBase64(работают с CLOB)
		getNodeValue/createTextNode(работают с CLOB)

[*] Функция getReleaseVersion, теперь возвращает версию XML.

[-] Исправлена ошибка в функции parseClob. Обработка файлов длиной больше 42K
    заканчивалась ошибкой "Unexpected end of file" 

v1.3

[+] Реализованы следующие функции:
		getParentNode, hasChildNodes, getLastChild, removeChild
		writeToFile(с параметром charset),
		writeToBuffer(с параметром charset и без),
		writeToClob(с параметром charset и без),
		parseClob
    Функции с параметром charset ожидают "стандартные" (не Oracle) названия	кодировок,
	например, Windows-1251, koi8-r и ISO-8859-5, а не CL8MSWIN1251 и CL8ISO8859P5.

v1.2

[*] В библиотеках для HPUX запрещено использование встроенных путей для
    поиска зависимых библиотек.

v1.1

[+] Сделана автоматическая инициализация.
	Теперь вызов initialize необходим только для получения ошибки инициализации. 

[+] Добавлена операция createEntityReference. 

[+] Атрибут standalone директивы <?xml ... ?> выводится только со
	значением "yes". C умолчательным значением "no" он опускается.

[-] В библиотеках для платформ TRU64 и HP-UX исправлены ошибки,
	которые возникали при возврате пустых строк. 

[-] В библиотеках для платформы HP-UX использован ключ линкера +s,
	который позволяет использовать переменную SHLIB_PATH для указания
	пути поиска зависимых библиотек. 

[-] В библиотеках для платформы SUN Solaris исправлена ошибка,
	которая приводила к сбою extproc при попытке открыть для записи
	недоступный файл.
