1. Данные шаблоны используются в АРМе "Администратор доступа" для
формирования следующих отчетов:

ua_obj.rdf 	   "Доступ к экземплярам системы"
ua_o_cat.rdf 	   "Доступ к экземплярам системы с категориями доступа"
ua_right.rdf 	   "Доступные ТБП, Операции, Представления, Переходы"
ua_group.rdf 	   "Список пользователей группы" и 
		   "Группы со своими пользователями"
ua_user.rdf 	   "Список групп, в которые включен пользователь" и
		   "Пользователи со своими группами"
ua_model.rdf 	   "Отчет о модели"
ua_obj_ex.rdf      "Доступ по реквизиту"
ua_admins.rdf      "Пользователи с правами администраторов"
ua_profiles.rdf    "Профили пользователей"
ua_temp_rights.rdf "Пользователи, временно получившие права"


Шаблоны представляют собой шаблоны отчетов Oracle Reports 6i и для
корректной работы требуют установки Oracle Reports Runtime 6i (Oracle
Reports Runtime можно установить из пакета "Клиентское место
"ЦФТ-Платформа развития").

2. Данные шаблоны используются АРМом "Администратор доступа" версии 69
и выше.
АРМ "Администратор доступа" версии 68 и ниже не может использовать эти
шаблоны.

3. АРМ "Администратор доступа" при работе с определенной прикладной
схемой использует шаблоны отчетов, которые находятся в каталоге,
указанном в параметре REPORTPATH схемы (значение этого параметра
можно посмотреть и изменить, используя АРМ "Администратор словаря
данных").

4. Начиная с выхода версии 69 АРМ "Администратор доступа" "старые" 
шаблоны отчетов в формате Crystal Reports (расширение .rpt) более 
не поддерживаются. Это означает, что никакие изменения и 
исправления вноситься в них не будут.
Следует осуществить переход на "новые" шаблоны отчетов.
Начиная с января 2006г. отчеты в "старом" формате Crystal вообще не
запускаются из АРМ "Администратор доступа".

5. Для того чтобы установить данные шаблоны отчетов, нужно проделать
следующие шаги.
5.1 На всех персональных компьютерах, на которых предполагается
использовать АРМ "Администратор доступа", установить Oracle Reports
Runtime 6i (Oracle Reports Runtime можно установить из пакета
"Клиентское место "ЦФТ-Платформа развития").
5.2 Используя АРМ "Администратор словаря данных", узнать значение
параметра REPORTPATH прикладной схемы.
Это каталог, в котором должны быть размещены шаблоны отчетов.
5.3 В каталог шаблонов скопировать файлы шаблонов отчетов (файлы с
расширением .rdf).

6. Начиная с 74 версии "Администратора доступа", настройка отчётов 
сделана на основе xml, с привязкой к конкретному контексту окна, 
и осуществляется в предопределённых тэгах. Для каждого контекста 
существует список параметров, передаваемых в отчёт: 
	1. Определённый, т.е. передаваемый всегда; 
	1.1 Конкретного контекста; 
	1.2 Необходимый для работы отчёта; 
	2. Неопределённый, т.е. дополнительные параметры отчёта, 
	подразумевающие тонкую настройку;
Все параметры могут быть переопределены. Наиболее приоритетными 
считаются параметры, содержащиеся в тэге report, менее приоритетными 
считаются параметры в тэге params, затем параметры, передаваемые 
из модуля "Администратор доступа". Параметры тэга params являются 
общими для всех отчётов данного контекста окна. 

Особенности: параметр SUBJ_TYPE, устанавливаемый для субъектов, имеет 
двойственную сущность, регулирует режим запуска отчёта, т.е. разрешает 
или запрещает запуск отчётов в зависимости от типа субъекта(ов). Также 
данный параметр передаётся в качестве определённого в текущем контексте.
