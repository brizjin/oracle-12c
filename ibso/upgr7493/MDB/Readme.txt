ВНИМАНИЕ !!!
Установка системных хранилищ должна производиться после установки
лицензионной информации. Инструкцию по установке см. в файле
DOC\license_setup.txt.
Для установки хранилищ необходимо использовать АРМ Администратор проектов
версии 6.93.0.109  (Pick93_109.exe) и выше.

       Системные классы, зависимые от версии ТЯ.

1. Хранилище SIMPLE.MDB содержит набор скалярных типов (даты,строки,тип "Время"),
которые используются в системных типах пп.2-8. Устанавливать это хранилище
следует до установки всех остальных хранилищ (пп.2-8).

2. Хранилище JOBS73.MDB содержит обновление типов "Выполнение заданий по
расписанию" (SYSTEM_JOBS) и "Выполнение текстовых заданий" (TEXT_JOBS). 
Эти типы необходимо установить после UPGRADE технологического ядра.

3. Хранилище RUN_STAT42.MDB содержит типы для сбора статистики о
выполнении пользовательских операций версии 4.2. Эти типы необходимо установить
после UPGRADE технологического ядра. Более подробное описание самого типа
и его установки содержится в файле run_stat.txt.

4. Хранилище DIARYS73.MDB содержит тип "Системные журналы" (SYS_DIARYS), 
который позволяет просматривать содержимое системных журналов.
Его необходимо установить после проведения UPGRADE. Установка возможна
в двух режимах - стандартном (по списку DIARYS_STD.PCK) или расширенном
(по списку DIARYS_ALL.PCK). Расширенный вариант дополнительно к стандартному
включает в себя набор представлений по журналам с фильтром по подразделению
пользователя.
Список элементов для удаления Del_diary_ext.xpck содержит набор дополнительных
системных журналов и может быть использован в случае, когда установка хранилища 
DIARYS73.MDB ошибочно была произведена в расширенном режиме.

5. Хранилище VALH70.MDB содержит библиотеку работы с "Историей изменения
реквизитов экземпляров" (операция HLIB в типе SYS_DIARYS) и примеры ее
использования (операции в типе USER - VALUES_HISTORY, VALUES_HISTORY_1).
Хранилище следует установить после проведения UPGRADE. Установка возможна
в двух режимах - стандартном (по списку VALH_STD.PCK - только библиотека
SYS_DIARYS.HLIB) или расширенном (по списку VALH_ALL.PCK - библиотека
SYS_DIARYS.HLIB и примеры ее использования USER.VALUES_HISTORY и 
USER.VALUES_HISTORY_1). Описание функций библиотеки содержится в файле valh.txt.

6. Хранилище META.MDB содержит метаклассы "ClassAttributes" (CLASS_ATTRIBUTES,
CLASS_ATTR_REF), "States" (STATES,STATE_REF),"MethodControls" (METHOD_CONTROLS,
METH_CONTROL_REF). Эти типы необходимо установить после UPGRADE технологического
ядра. Если тип STATES использовался в прежних версиях, то ПЕРЕД установкой
хранилища следует ВНИМАТЕЛЬНО изучить файл Meta.txt, в котором описаны основные
отличия новой версии от старой.

7. Хранилище RUNTIME.MDB содержит библиотеку работы с календарем (CALENDAR) и 
библиотеку работы с логическими блокировками (RESOURCE_MGR).
Его необходимо установить после проведения UPGRADE.

   7.1 Если на схеме установлен дистрибутив ЭДО, то необходимо получить его версию выполнив следующий
   запрос на схеме владельца:
	select edoc_mgr.get_version from dual.
   Функция возвращает версию дистрибутива ЭДО. 
   Если текущая версия ТЯ не попадает в диапазон версий, соответствующих текущей версии ЭДО, то
   необходимо восстановить оригинальную версию библиотеки RUNTIME.EDOC из дистрибутива ЭДО, установив
   хранилище EDOC.mdb.
   Соответствие версий RUNTIME.EDOC между дистрибутивом ЭДО и Серверной части ТЯ:
	1. Дистрибутив ЭДО версии 1.18.0.6 доступен при использовании Серверной части ТЯ версии выше 7.4.9.0.
	2. Дистрибутив ЭДО версии 1.17.0.2 доступен при использовании Серверной части ТЯ версий 7.4.8.2 - 7.4.9.0
	включительно.

8. Хранилище EVENTS72.MDB содержит типы "Системные события" (SYS_EVENTS)
и "Справочник событий" (EVENTS), которые позволяют инициализировать
прикладные события и обрабатывать их. Если эти типы предполагается
использовать, то их необходимо обновить после UPGRADE. После установки
хранилища нужно удалить индекс IDX_SYS_EVENTS_STATUS_PRIOR_NN (Events72_Del.xpck прилагается).
Вместо него создается другой индекс.

9. Хранилище AccessInfo.mdb содержит набор типов, операций и представлений
для отчетов по системе доступа.
В AccessInfo.mdb объединены следующие изменения:
а) Реструктурирован тип "Субъекты доступа" (USERS), поэтому добавлен сам
тип, и операции других типов, которые ссылаются на его представления.
б) Изменены представления "Система доступа (группы)" (VW_CRIT_USER_GROUPS) и
"Система доступа (пользователи)" (VW_CRIT_USER_USERS) ТБП "Пользователи" (USER).
в) Переведены в формат Oracle Reports следующие отчеты ТБП "Пользователи"
(USER): "Доступ. Список пользователей группы" (OP_UA_GROUP) и
"Доступ. Список групп, в которые включен пользователь" (OP_UA_USER).

Внимание!!!
Если представления VW_CRIT_EQUAL_SUBJ, VW_CRIT_GROUPS, VW_CRIT_SUBJ_EQUAL и 
VW_CRIT_USERS не были удалены из ТБП "Субъекты доступа" (USERS), то перед 
установкой хранилища AccessInfo.mdb необходимо с помощью 
АРМа "Администратор проектов" удалить объекты по списку DelObjAI.pck 

Шаблоны отчетов по системе доступа в формате Oracle Reports находятся
в подкаталоге REPORTS. Их следует скопировать в каталог, откуда
выполняются все отчеты системы.

10. Список Delobjai_rep.pck содержит устаревшие операции-отчеты
по системе доступа. Рекомендуется с помощью АРМа Администратор проектов
удалить операции по списку Delobjai_rep.pck, а также удалить соответствующие
шаблоны отчетов в каталоге для отчетов схемы (файлы UA_USER.RPT и UA_RIGHT.RPT).

11. Хранилище files.mdb содержит системные типы данных для поддержки
экспорта-импорта файлов с клиента на сервер и наоборот. Если поддержка
экспорта-импорта файлов в Навигаторе не была установлена ранее, тогда
нужно установить все объекты из указанного хранилища.

12. Хранилище recall_cfg.mdb содержит в себе функционал перевызова операций в платформе 2MCA.
Его необходимо установить после проведения UPGRADE.

13. Хранилище SLA.mdb содержит систему отслеживания параметров отклика в платформе 2MCA.
Его необходимо установить после проведения UPGRADE.

14. Хранилище JobWatcher.mdb содержит сервис контроля серверов заданий 2 MCA.
Его необходимо установить после проведения UPGRADE.

15. Хранилище FS.mdb содержит системные типы данных "ЦФТ - Подсистемы работы с формами".
Если исходная версия ТЯ ниже 7.4.8.0 обязательно необходимо: 1) установить хранилище;
2) запустить файл patch_fs.bat с указанием в качестве параметра коннект-строки к
схеме-владельцу для раздачи привилегий пользователю-посреднику для работы Сервера
Приложений 2 МСА в режиме эмуляции Платформы 1.
