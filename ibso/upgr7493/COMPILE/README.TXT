Набор сценариев для пакетной компиляции операций 
системы.

int_cls.sql  - создание интерфейсных пакетов классов;
intcls.sql   - пересоздание схемы хранения классов;
objs.sql     - перестроение OBJECTS, COL2OBJ, COLLECTIONS; 
int_meth.sql - создание интерфейсных пакетов операций;
c_meth.sql - компиляция операций - создает исполняемый
      скрипт, затем запускает его;
u_meth.sql - дополнительный скрипт для c_meth.sql, в нем
      можно модифицировать условие для выборки операций;
c_method.sql - компиляция операций без промежуточного
      исполняемого скрипта;
u_method.sql - дополнительный скрипт для c_method.sql, в нем
      можно модифицировать условие для выборки операций;
c_clear.sql - удаление лишних интерфейсных пакетов
      классов, операций, очистка таблицы sources;
u_clear.sql - дополнительный скрипт для c_clear.sql;
nnfk_idx.sql - создание ограничений целостности и индексов по
      ссылкам и коллекциям.
Примечание. Скрипты запрашивают имя PIPE для вывода текущей 
      отладочной информации.

errfrm.sql - анализирует формы операций на соотвествие контролов
      параметрам/переменным операции и выводит найденные ошибки;
errctl.sql - анализирует формы операций на уникальность имен контролов
      и выводит найденные ошибки;
errcls.sql - выводит ошибки в интерфейсных пакетах классов;
errcrit.sql - вывод сообщений об ошибках компиляции представлений;
c_pack.sql - компилирует инвалидные пакеты (alter package
      compile body), условие выборки можно модифицировать
      прямо в нем;
c_obj.sql - компилирует инвалидные объекты (alter ... compile);
m_errors.sql - вывод сообщений об ошибках компиляции операций,
      условиe выборки можно модифицировать прямо в нем;
m_warn.sql - вывод сообщений о предупреждениях компиляции операций,
      условиe выборки можно модифицировать прямо в нем;
build_fk.sql - восстановление иерархических ограничений целостности
      (вывод в пайпу DEBUG).
others.sql - перестроение представлений;
dropidx.sql- удаление пользовательских индексов с формированием скрипта
      по их созданию (сам скрипт пишет в пайпу DEBUG, а сгенеренный
      пишет в DEBUG2).
с_reb.sql  - перестроение таблиц типов (вывод в пайпу DEBUG).
c_stor.sql - запускалка для nnfkidx (вывод в пайпу DEBUG1).
с_stat.sql - пересчет статистики (вывод в пайпу DEBUG).
сstat.sql  - пересчет статистики под Oracle8 (вывод в пайпу DEBUG).
Примечание. Скрипт cstat.sql принимает command-line параметр, 
    который содержит команду SQL*Plus, которая будет выполняться перед 
    запуском сгенеренного скрипта (например, можно задать pause, 
    чтобы иметь возможность подредактировать сгенеренный скрипт перед 
    его запуском).
