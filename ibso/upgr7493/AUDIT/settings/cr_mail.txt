Cкрипты инициализации системы Активного Аудита
Необходимо выполнять на схеме AUD.
------------------------------------------------------------------------------
1. cr_mail_settings.sql - инициализация настроек для активного аудита.

В процессе выполнения необходимо указать :

Email SMTP server [&&SMTPSRV]: 
Указать: Имя smtp сервера, через который будут отсылаться оповещения
о событиях, происходящих в системе.
Внимание! Должна быть обеспечена возможность установить TCP/IP соединение
с этим сервером с машины сервера БД.

Encoding characterset [&&ENCODING]: 
Указать: Кодировка почтовых сообщений (по умолчанию KOI8-R)

Default sender email [&&SENDER]:
Указать: Желаемый адрес отправителя, который будет указан
в письме-оповещении.
По-умолчанию будет указан AUD@AUD.

Default sender name [&&SENDER]:
Указать: Желаемое имя отправителя, которое будет указано
в письме-оповещении.
По-умолчанию - не указано.

Default email auth [&&AUTH]:
Указать: Нужна ли авторизация на SMTP сервере(YES/NO)
По-умолчанию будет указан NO.

Default email user [&&SUSER]:
Указать: Желаемое имя пользователя для авторизации
По-умолчанию - не указано.

Default email pass [&&PASS]:
Указать: Желаемый пароль для авторизации
По-умолчанию - не указано
------------------------------------------------------------------------------

2. cr_mail_note.sql - добавление Email событий-нотификаций.

Добавляет события:
- LOGON_SUCCESS	Событие входа привилегированным пользователем
- LOGON_ERROR	Событие ошибки входа привилегированным пользователем
- LOGON_WARNING	Отчёт о неуспешных входах
- EDOC_BAD_PKDB	Событие ошибки проверки целостности БОК
и получателей для этих событий.
События добавляются со статусом указанным, пользователем. Получатели добавляются 
со статусом ACTIVE.


Использует настройки, выполненные скриптом  cr_mail_settings.sql.

В процессе выполнения необходимо указать :

Main recipient email [&&RECPT]:
Указать: Адрес получателя писем-оповещений. Рекомендуется указывать сразу
группу подписки. По умолчанию будет указан Revisor@AUD.

Main recipient name [&&RECPT]:
Указать: Имя получателя писем-оповещений. Может быть не указано.

Status for CONNECT notifications [&&STATUS]:
Указать: ACTIVE

Status for EDOC_BAD_PKDB notification [&&STATUS]:
Указать: ACTIVE
------------------------------------------------------------------------------

3. cr_mail.sql - последовательно выполняются действия выполняемые скриптами 
cr_mail_settings.sql, cr_mail_note.sql:

  - инициализация настроек для активного аудита.
  - добавление Email событий-нотификаций.

Запрашиваемые параметры  - см. описания к скриптам в пп.1,2
