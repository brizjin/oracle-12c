После  апгрейда  ТЯ  автоматически  создается задание Oracle для сбора
расчета   фактических   значений   лицензионных  ограничений  и  сбора
лицензионного  отчета, время запуска этого служебного задания - 3 часа
ночи, периодичность - 1 раз в сутки.

Во  время  выполнения  служебного  задания  для  расчета  лицензионных
настроек  на  схеме  недопустимо  производить  останов схемы (например, 
для   создания  бэкапа) и  установку лицензионной информации с помощью 
АРМа "Администратор дистрибутива" (при  непосредственном  запуске АРМа 
или при его запуске в безынтерфейсном режиме, в  частности при запуске
из приложения CFTUpdate).  Иначе  лицензионные  настройки  могут  быть 
рассчитаны  с  ошибками  (например, ORA-01089: никакие  действия  не 
разрешены,  т.к.  идет срочный  останов), что повлияет на доступность 
элементов в АРМе "ЦФТ-Навигатор".


Рекомендуется настроить время  запуска  служебного  задания для сбора
лицензионного отчета таким образом, чтобы во время выполнения задания
не запускались процедуры, требующие останова схемы.
 
Изменить  время  запуска  системного  задания можно с помощью операции
"Запуск  системных  заданий" в ТБП "Выполнение заданий по расписанию".
На  экранной  форме  операции  надо  будет  указать  параметры задания
проверки  лицензионной информации - т.е. включить флаг "Запуск задания
проверки лицензионной информации" и установить требуемое время.

Внимание!!!
Должно  быть  установлено  хранилище JobsXX.mdb из состава апгрейда ТЯ.
