Краткая инструкция по проведению апгрейда ТЯ.

Подробное описание для выполнения отдельных пунктов апгрейда ТЯ смотрите 
в файле DOC\readme_upgrade.txt в разделе "Процедура проведения UPGRADE".

В каталоге PATCH находятся скрипты для установки последних изменений ТЯ. 
Если уже установлена ТЯ версии 7.3.2.1 или выше, то полностью выполнять весь 
UPGRADE не обязательно, достаточно выполнить патч до текущей версии.
Порядок и условия установки патча описан в файле DOC\patch.txt.


1. Ознакомьтесь с требованиями к серверной платформе и клиентской станции 
(см. пункты 1,2 DOC\readme_upgrade.txt).

2. Проведите апгрейд схемы ревизора (AUD), менеджера аудита (AUDM),
предоставление привилегий схеме-владельцу IBSO с помощью FIRST.BAT
(см. пункт 3 DOC\readme_upgrade.txt).

3. Проведите апгрейд схемы владельца IBSO с помощью одного из исполняемых файлов
(подробнее см. пункты 4-8 DOC\readme_upgrade.txt):
   UPGR_.BAT - режим перекомпиляции модели (типовой режим), 
   UPGR.BAT  - режим перекомпиляции модели с перестроением таблиц,
   UP_.BAT   - многопоточный режим перекомпиляции модели,
   UP.BAT    - многопоточный режим перекомпиляции модели с перестроением таблиц.

(!) Обратите внимание на требование к версии JAVA-компилятора в случае, если 
    на схеме используется продукт Платформа 2 MCA.

(!) Обратите внимание на режим проведения апгрейда - "без вопросов", а 
    также на настройку автоматического запуска мониторов коммуникационного 
    канала.

(!) Начиная с ТЯ версии 7.3, в апгрейд введены параметры :
    - значение настройки CREATE_SYNONYMS умолчательного профиля DEFAULT;
    - минимальный период хранения (в днях) результатов выполнения отчетов
      в очереди отчетов (обрабатываемой агентом сервера отчетов - ORSA);
    - признак пересоздания пользовательских ролей. 
 
(!) Изменено значение по умолчанию параметра апгрейда  "Признак замены у 
    пользователей прямых грантов на пакеты, используемых в отчетах Oracle 
    Reports".


4. Перезапустите задание менеджера аудита (AUDM) с помощью LAST.BAT (пункт 9 
   DOC\readme_upgrade.txt).

5. Сохраните логи апгрейда с помощью getlog.bat (пункт 10 
   DOC\readme_upgrade.txt).

6. Проанализируйте логи апгрейда, используя рекомендации в файле 
   DOC\readme_upgrade.txt.

7. Установите новые версии используемых внешних библиотек (пункты 11-14 
   DOC\readme_upgrade.txt).

8. Установите лицензионную информацию (пункт 15 DOC\readme_upgrade.txt).

9. Установите системные хранилища из каталога MDB (пункт 16 
   DOC\readme_upgrade.txt).
   - рекомендуется проверить, что периодичность запуска задания "Обновление
системных пайп" находится в интервале от 120 до 300 минут.
