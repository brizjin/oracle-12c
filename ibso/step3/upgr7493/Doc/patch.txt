Узнать версию технологического ядра можно выполнив следующий запрос
на схеме владельца: 
	select inst_info.get_version from dual 
Если функция возвращает другое значение, то патч устанавливать нельзя - 
необходимо выполнить полностью весь UPGRADE (см. upgrade.txt)

С версии ТЯ 7.3.1.0 внешние процессы LOCK_INFO не используются. 
(см. PLPLUS\lockinfo.txt).

1. Ознакомьтесь с требованиями к серверной платформе и клиентской станции. 
   (см. пункты 1,2 readme_patch.txt).

2. Выполните раздачу необходимых привилегий пользователю-владельцу схемы IBSO.
   Для этого необходимо запустить файл patch_sys.bat (см. пункт 3 
   readme_patch.txt). Если данного скрипта в патче нет, то этот пункт можно
   пропустить.   

3. Проведите обновление схемы менеджера аудита (AUDM).
   Для этого необходимо запустить файл patch_audm.bat (см. пункт 4 
   readme_patch.txt). Если данного скрипта в патче нет, то этот пункт можно
   пропустить.   

4. Проведите обновление схемы ревизора (AUD). 
   Для этого необходимо запустить файл patch_aud.bat (см. пункт 5 
   readme_patch.txt). Если данного скрипта в патче нет, то этот пункт можно
   пропустить.   

5. Проведите обновление схемы владельца IBSO с помощью исполняемого файла
   patch_ibs.bat (см. пункт 6 readme_patch.txt).
   Если данного скрипта в патче нет, то этот пункт можно пропустить.   

(!) Обратите внимание на требование к версии JAVA-компилятора в случае, если 
    на схеме используется продукт Платформа 2 MCA.

(!) Обратите внимание на режим проведения обновления - "без вопросов".

6. Проанализируйте логи апгрейда, используя рекомендации в файле 
   readme_patch.txt.

7. Если в папке PATCH существует файл core*.xsap, то установите лицензионную
   информацию (см. пункт 8 readme_patch.txt).

8. Установите системные хранилища из каталога MDB в корневой папке, согласно
   описанию установки хранилищ с системными типами в файле PATCH\MDB\readme.txt
   (см. пункт 10 readme_patch.txt). 
   Если данного файла нет, то этот пункт можно пропустить.
